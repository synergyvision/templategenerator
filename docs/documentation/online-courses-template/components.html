Components
global

Este directorio contiene componentes que se utilizan repetidamente a lo largo de la plantilla, y pueden ser reutilizados para su modificación

header.html

Este componente incluye todo lo referente al encabezado del sitio.

Todos los datos mostrados son extraídos del directorio "_data", archivo "header.yml"

header-links.html

Este componente muestra un link, y despliega un menu al hacer "hover" sobre ese, si y solo si contiene una lista que desplegar

Parámetros:

svg-directory: Directorio donde se encuentran los íconos utilizados

link: Objeto que contiene "icon", ícono opcional que se encuentra lateral al nombre; y "name", nombre del ícono

sections: Arreglo que contiene un "url" de redirección, y un "title" título del link

position: "left" si el menú desplegable debe mostrarse hacia la izquierda del link, "right" si a la derecha.

footer.html

Este componente incluye todo el pie de página del sitio.

Todos los datos mostrados son extraídos del directorio "_data", archivo "footer.yml"

title.html

Este componente renderiza el título principal de la platilla, separando el título y subtítulo por una línea.

Parámetros:

title: título a mostrar

subtitle: subtítulo a mostrar. Opcional

size: "medium", si desea disminuir el tamaño del título.

search-input.html

Este componente muestra un buscador customizado

Parámetros:

placeholder: texto por defecto en el buscador

login-input.html

Este componente muestra un input customizado, con un borde inferior del color primario del sitio

Parámetros:

label: nombre del input.

input-type: Si desea un campo para una descripcion seleccione "textarea", si desea un campo ligero indique el tipo del campo, como "number", "text", etc

input-name: Texto que desea mostrar por defecto en el campo

id: identificador del campo

input-icon: id del ícono svg

atts: texto con todos los atributos extra que desea añadir al input. Opcional

svg-directory: directorio donde se encuentran los íconos svg

button.html

Este componente renderiza el botón pricipal de la plantilla, que se reutiliza a lo largo de esta.

Parámetros:

btn: Objeto que incluye : "href", nombre que indica la redirección, "type", indica si es "primary" el cual pinta el botón del color primario, o "secondary" en el cual se invierten los colores y "text", incluye el texto del botón.

icon: nombre del ícono svg. Opcional. Renderiza un ícono a la derecha del texto

onclick: nombre de la función a la que desea llamar de ser requerido. Opcional.

button-secondary.html

Este componente renderiza un boton, con bordes redondeados y del color primario del sitio, con un ícono svg en el centro. NO incluye texto

Parámetros:

href: redirección del botón/ acción.

directory: directorio donde se aloja el ícono svg.

icon: id del ícono svg.

checkbox.html

Este componente renderiza un checkbox customizado.

Parámetros:

value: valor del input. Opcional

nombre del input. Opcional

id: identificador del input

click: nombre de la funcion que se debe ejecutar al hacer el check. Opcional

checked: Ponga en true si desea que checkear el input por defecto, false si no

text: Texto incluido en el lado derecho del check. Opcional

banner-secondary.html

Este componente renderiza un banner con una imagen de fondo y un titulo alineado a la derecha.

Parámetros:

title: texto que se muestra en la exquina derecha del banner.

background: nombre de la image de fondo

conversation.html

Este componente renderiza un chat, que incluye un header con la persona que se conversa y su información, el contenido del chat, y un input para escribir.

Parámetros:

size: tamaño del chat. "small", si desea un chat pequeño. Por defecto el chat es de tamaño grande.

chat: Objeto que incluye la información dinámica del chat. Incluye "image", objeto que tiene un "name" nombre de la imagen y un "alt", texto alternativo a la imagen; "name", nombre de la persona con la que se conversa; "status": status de la persona y "chat", arreglo que contiene la conversación en sí. El arreglo "chat" incluye el atributo "received" si el mensaje es recibido o "sent" si el mensaje fue enviado, y "time" que indica la fecha de envío del mensaje

chat-data: Objeto que incluye la información estática del chat. Incluye un "header" que contiene "title", puede ser el prefijo del titulo de la conversa; "message-bottom" que contiene un "title" siendo este el título de la hora del mensaje enviado; "text-area" que contiene los parámetros del componente "login-input" ; y "btn-send" que incluye los parámetros utilizados en el componente "button".

chat.html

Este componente renderiza un chat, lo que incluye un buscador con lista de usuarios y una conversación en la ezquina derecha (componente conversation.html).

Los datos utilizados en est componentes estan inluidos en el directorio "_data", archivo "conversations"

Parámetros:

type: Lista de usuarios en una conversación. Colección que incluye "image" conteniendo "name", el nombre de la imagen, y "alt" texto alternativo a la imagen; y "name" nombre de la persona.

image-directory: dirección en la que se encuentran las imagenes utulizadas

empty-message.html

Este componente es utilizado para los mensajes de vacío, contiene un texto y un ícono, y si se desea un botón de redirección.

Parámetros:

message: Mensaje que se desea mostrar

icon: id del ícono svg que se desea mostrar

btn: Objeto que incluye los parámetros del componente "button"

svg-directory: dirección donde se encuentra alojado el ícono svg.

message.html

Este componente muestra un mensaje informativo, con un ícono de alerta

Parámetros:

info: texto de información a mostrar

horizontal-menu.html

Este componente consiste de un menú horizontal, que permite cambiar el contenido inferior

Parámetros:

menu: Arreglo que contiene un "link", o nombre de la sección, y un "component" que especifica la dirección donde se encuentra el componente (contenido a mostrar).

🚨 Los componentes listados NO pueden tener parámetros

fields.html

Este componente nos permite incluir la cantidad de inputs, checkbox, mensaje de información o componente que se desee a un formulario, especificando su tipo

Parámetros:

inputs: Arreglo que contiene : 1. "title", muestra un titulo centrado y es opcional, 2. "type", que puede ser "input", "checkbox", "info" o "component" .

Si el "type" es igual a "input" es necesario el atributo "label" que muestra el nombre del input, un objeto "input" que contiene los parámetros del componente "login-input" y el parámetro "data". Este parámetro "data" incluye un arreglo que debe contener el atributo "name", de manera que el name de los campos coincidan con el name del "input" para poder mostrar cada dato en el input correspondiente.

Si el "type es igual a "checkbox" , es necesario el atributo "checked" y "text" ( parametros del componente "checkbox")

Si el "type" es igual a "info", es necesario el atributo "text-info" (llamado "info" en el componente message.html), y el atributo "type".

Si el "type" es igual a "component", es necesario un atributo name, que hace referencia al lugar donde se aloja este componente. 🚨 Es importante saber que el componente aquí pasado NO puede tener parámetros.

image-title.html

Este componente muestra un título alargado, con una imagen en el extremo izquierdo

Parámetros:

info: Objeto que incluye: "image", que contiene un "name" o nombre de la imagen, y un "alt" o texto alternativo a la imagen; "title" o título a mostrar; y "subtitle" o subtitulo a mostrar, alternativo.

message-fixed.html

Componente que incluye un widget fijato en la ezquina inferior derecha, accionando al ser clickeado el componente coversation

numbers-stripe.html

Componente que consiste en una franja, con cantidades

Parámetros:

numbers: Colección que contiene "number" o cantidad a mostrar, y "text" o texto a mostrar

selector.html

Este componente se comporta como un selector, incluyendo dentro de si las diferentes opciones a seleccionar. Ejemplo de esto, el selector de idiomas en el pie de página de la plantilla.

Parámetros:

options: Colección que incluye "name" siendo el nombre de la opción, y "default", que debe ser "true" si desea mostrar dicha opción por defecto.

story-stripe.html

Este componente muestra una sección básica, mostrando a un lado una imagen, y de otro un texto incluyento título.

Parámetros:

image-directory: directorio donde se alojan las imagenes mostradas

item: Objeto que incluye 1. "image", que contiene "name" o nombre de la imagen, y "alt" o texto alternativo a imagen; 2. "inverted", que se debe igualar a "true" si desea que el invertir los lados de la sección, 3. "title", que muestra el título de la sección, y 4. "text", que muestrael texto respectivo.

add-to-list.html

Este componente permite añadir un elemento a la lista, a través de un "input" y in boton de agregar.

Parámetros:

title: título de la lista

input: Objeto que contiene los parámetros del componente "login-input", además de un "btn", que contiene los parámetros del componente "button"

field-btns: Colección que incluye "name". Estos botones planos son mostrados en la fila de cada elemento añadido a la lista

add-to-list-group.html

Este componente permite añadir secciones y modulos respectivamante a un acordión

Parámetros:

input: Objeto que contiene los parámetros del componente "login-input"

btn: Colección que incluye los parámetros del componente "button"

field-btns: Colección que incluye "name". Estos botones planos son mostrados en la fila de cada elemento añadido a la lista

accordion.html

Este acordión es utilizado para modificar las secciones y módulos de un curso.

Parámetros

edit-info: incluye un "input", objeto que contiene los parámetros del componente "login-input"; y "btn", objeto que contiene los parámetros necesarios en el componente "button"

sections: Colección que contiene un "title", título mostrado en el encabezado del acordión, y otra colección llamada "modules" que representa los elementos de una sección.

Cada arreglo "modules", contiene un "title", título del módulo, y de quererse un "icon", donde se especifica el "href", que hace referencia al id del modal que desea abrir; y un "icon", que especifica el nombre del ícono svg

video-text-stripe.html

Este componente muestra una franja horizontal con un video o imagen que cubre la mitad (izquierda o derecha) de la franja, y un texto en el lado contrario. El fondo es degradado con los colores primarios del sitio.

Parámetros:

inverted: Si desea que el video/imagen se encuentren del lado derecho, ponga en "true"

video: Incluya si desea añadir un video. Objeto que incluye "source" para referir al nombre del video; y "external" que debe poner en true si el video viene de una fuente externa como Youtube

source-video: Si desea añadir un video, indique aquí el directorio donde se aloja (de ser un video local).

image: Incluya si desea añadir una imagen. Objeto que incluye "name" que hace referencia al nombre de la imagen, y "alt" o texto alternativo a la imagen

title: título de la sección, que utiliza el componente title.html

subtitle: subtítulo de la sección

vertical-card-menu.html

Este componente consiste en un box , que contiene un selector en el extremo izquierdo, que nos permite cambiar el contenido de este box.

Parámetros:

data: arreglo que contendrá los datos que deberán ser mostrados en los campos respectivos, por lo que deberá tener un atributo "name" que haga referencia al nombre o "name" del "login-input"

sections: Arreglo que contiene: 1. "title", título de la sección, 2. atributo que puede ser "one-side" si desea el contenido en todo el ancho del componente, "left-fields" si lo desea del lado izquiero del componente, "right-fields" si los desea del lado derecho del componente, "left-fields-big" si los desea del lado izquierdo y ocupando 3/4 del ancho, "left-fields-small" si lo desea del lado izquierdo y 1/4 del ancho, "right-fields-big" si lo desea del lado derecho y 3/4 del ancho del componente, y "right-fields-small" si lo desea del lado derecho del componente ocupando 1/4 del ancho del mismo; y 3. "btn" que posee un arreglo con los parámetros del componente "button"

🚨 Cada objeto de (2.) es un arreglo. Cada posición de este arreglo debe cumplir con los atributos especificados en el componente fields.html dependiendo de lo que se desee (información, checkbox, input o componente). Se recomienda ver el componente field.html

Toda la información mostrada en esta sección es extraida del directorio "_data", en el archivo "home.yml", objeto "features"

feature-stripe.html

Toda la información mostrada en esta sección es extraida del directorio "_data", en el archivo "home.yml", objeto "features"

section-courses.html

Este componente consiste en una sección que incluye título, y un carousel con cartas de presentación de los cursos, contenidos en un carousel

title: Nombre que se le desea dar a la sección

id: Si desea añadir más secciones para mostrar diferentes grupos de cursos, es importante variar dicho id para cada sección. Hace referencia al id del carousel.

section-contact.html

section-community.html

Esta sección, muestra un título (componente title.html)), y en la parte inferior tantos "community-card" como lo indique la colección

Toda la información mostrada en esta sección es extraida del directorio "_data", en el archivo "home.yml", objeto "team"

Cards

Este directorio contiene diferentes "Cards" utilizados a lo largo de la plantilla, y pueden ser reutilizados y modificados posteriormente.

Blog-card.html

A blog card that displays the main information of the blog.

Parameters

url: The path to redirect

image-directory: The image source directory

image: Object wich include: name, the name of tha image. alt: Alternative text to image

title: The title of the blog post

description: Breve descriptionof the blog post

buy-card.html

This card includes the payment information, including prices, discount prices, coupons etc.

Parameters:

pay-info: Object wich includes: text, the title of the card, currency, the current currency, percentage-off-text, wich displays an "off", coupon-input wich include the parameters of an input, and a btn, wich includes the parameter of a button component

price: the current toal price

old-price: the total price without discount

svg-directory: source of the svg icon

category-card.html

This component shows a rounded button

Parameters:

name: Displays the title of the button

href: path to redirect

community-card.html

This component displays an element with a drop shape, and an image inside

Parameters:

member: Object that includes a name, a text (short text), and an image, wich includes a name (name of the image), and an alt (alternative to the image)

image-directory: The image directory

course-card.html

Parameters:

card: Object wich includes a name, owner, image (wich includes a name, and an alt), score (number between 0 and 5), discount (can be true or false), currency, discount-price (if discount equal to true), and an url to redirect

bought-info: Displays the progress of the course. Optional. Object wich includes completed (the percentage of completion), rating ( number between 0 and 5)

added: Set to true if you want to display the added buttons

course-feature-card.html

This component displays a card with listed texts

Parameters:

title: Displays the name of the card

section: Array that includes, an icon, and a text

svg-directory: The path of the svg icons

course-membered-questions.html

This components displays a card a question card, including the person image, the question and the description.

Parameters:

question: Object that includes a question (the question by it self), an answer, and a person with an image, wich includes a name and an alt (alternative text to image)

credit-card-edit.html

This component display a card with the credit card information.

Parameters:

This component has no parameters, so the information is taken from the data directory

instructor-card.html

This component displays a breve profile of an instructor.

Parameters:

title: displays the title of the card

person: Object that includes a name, and image (wich includes a name, and an alt), a short-description of the instructor, features (an array wich includes an icon and a text), and a href wich redirect to a page

image-directory: directory of the instructor image

btn-more: Object that includes the button component parameters

price-card.html

This component displays a buy card for a course, showing the prices and a list of buttons.

Parameters:

price: The price of the course

discount-price: The price after the discount

buttons: An array wich includes an href, and an icon

svg-directory: the directory of the svg icons

students-review-card.html

This component displays a card wich shows a person review of a course

Parameters:

student-review: Object that includes a name, a text, an image wich includes a name, and an alt (alternative to the image), an score (a number between and 5), a date,

image-directory: the directory of the image

switch.html

This component shows a simple wich displays a text in the left side.

Parameters:

text: Text to display

course-detail-pages

Este directorio contiene cada una de las vistas del detalle de un curso.

course-detail-instructor.html

Este componente muestra la vista de un curso, desde la perspectiva del instructor, permitiendo la modificación del mismo. Incluye lista de estudiante, secciones, información básica, etc.

Parámetros:

Este componente no posee parámetros pues utiliza la información del curso (página actual)

course-detail-subscribed.html

Este componente muestra el detalle de un curso, cuando el usuario ya se encuentra suscrito en el mismo, mostrando cada una de las secciones que contiene el curso, la descripción, preguntas, campo para realizar una nueva pregunta, y conversación con el instructor.

Parámetros:

Cada una de estas secciones pertenece a un componente particular, por lo que se utiliza el parámetro "page", que hace referencia a la página actual, es decir, la información del curso (almacenada en los archivos ".md" del directorio "_online-courses-courses") , y esta a su vez es pasada a los demás componentes segun sea necesario.

course-detail.html

Este componente muestra el detalle de un curso al que el usuario no se ha inscrito. Mostrando información relevante del curso como precio, descuento, video introductorio, qué incluye el curso , cuáles son las aptitudes que desarrollarán los estudiantes, "feedback" de los estudiantes, descripción escrita, e información básica del instructor que la dicta.

Parámetros:

Cada una de estas secciones pertenece a un componente particular, por lo que se utiliza el parámetro "page", que hace referencia a la página actual, es decir, la información del curso (almacenada en los archivos ".md" del directorio "_online-courses-courses") , y esta a su vez es pasada a los demás componentes segun sea necesario.

course-sections

Este directorio contiene varios de los componentes utilizados en el detalle de un curso, tanto en la vista de estudiantes inscritos, como en la de no inscritos.

my-courses.html

Este componente itera sobre la coleccion de cursos situada en "_data/user", donde se especifican los id de los cursos en los que está inscrito el usuario, y los compara con la colección de cursos, para renderizar aquellos que se encuentren en la lista

Incluye paginado, de ser la cantidad de cursos mayores a 9.

Muestra un mensaje de vacío (componente empty-message.html), de no haber cursos en la colección

🚨 Si desea añadir algun curso al usuario, añada al arreglo de courses en "user.yml" el id del curso que desee

course-questions.html

Este componente se ha utilizado en la plantilla para mostrar la sección de preguntas/respuestas en el detalle del curso, cuando se encuentra isncrito a éste. Consta de una sección de "Cards" paginados, de preguntas y respuestas, mostrando el total de preguntas realizadas, y un botón, que al ser seleccionado muestra un formulario para hacer una nueva pregunta.

Parámetros:

Este componente utiliza igualmente la informacion particular de un curso, nombrada "page", y la información estática del archivo "courses.yml"

Este componente utiliza clases de "css" para distribuir los diferentes componentes que tiene dentro de sí.

students-review.html

Este componente se ha utilizado en la plantilla, especificamente en el detalle de un curso, cuando el usuario no se ha inscrito en el curso, para mostrar la sección de "feedbacks" de los estudiantes

Parámetros:

title: muestra el titulo de la sección

students-reviews: recibe una colección con la información que necesita el card "students-review-card.html" , mencionado en los componentes anteriores

Este componente incluye tambien un paginado , encerrando cada uno de los "cards" con la clase "card-cont"

course-modules-dropdown.html

Este componente consiste en un menu fijado en la ezquina derecha, conteniendo secciones desplegables con módulos dentro de estos, que pueden ser marcados con un "check"

Parámetros:

page: Incluye la información almacenada en la metadata del curso (archivo ".md" incluído en el directorio _online-courses-template), utilizando el "course-content" que ahí se encuentra, siendo este una colección que incluye: "title", título de la sección, y "modules", un arreglo que contiene "title", título del módulo.

course-description.html

Este componente muesta en texto plano una descripción

Parámetros

title: Incluye el titulo a mostrar en la descripción

description: colección que incluye un "title", subtitulo del parrafo si se desea, y un "paragraph", texto del párrafo.

ask-question.html

Este componente muestra los "inputs", para realizar una nueva pregunta. Este componente se incluye en "course-questions.html"

Este componente utiliza la información alojada en el directorio "_data", archivo "courses", en la sección de detalle de curso

instructor

En este directorio se encuentran algunos de los componentes utilizados en el dashboard del instructor.

side-nav.html

Este componente consiste en un menú lateral, mostrado en el dashboard del instructor en la pantilla

Este componente utiliza la información alojada en el directorio "_data", archivo "instructor", en el objeto "side-nav"

Consiste en un arreglo que contiene texto e íconos y una imagen de perfíl.

create-course-stepper.html

Este componente consiste en un paso a paso, que indica el paso en el que se encuentra.

Incluye un encabezado, donde se marcan dichos pasos, un contenido, y un "footer"

Este componente utiliza la información alojada en el directorio "_data", archivo "instructor", en el objeto "dashboard-create-course"

🚧 Si desea añadir los pasos para la creación de un curso, agrege un nuevo "tab" al contenido. Si desea modificar alguno de los pasos, modifique o suplante el componente en el "tab" correspondiente

create-basic-info.html

Este componente consiste en un formulario básico con información definida. Incluye dos campos a la izquierda de este, y uno a la derecha.

Parámetros:

info: Objeto que incluye "name", "description", y "price", cada uno de estos con los parámetros utilizaados por el componente "login-input"

edit-module.html

Este componente se utilizó y fue pasado por parámetro al modal de edición del módulo de un curso

🚧 Si desea editar este modal, cambie este componente!

Este componente utiliza la información alojada en el directorio "_data", archivo "instructor", en el objeto "sections / popup-edit"

includes-course-table.html

Este componente es pasado a uno de las secciones de "vertical-card-menu.html". Revise la documentación del componente

Es utilizado en la plantilla para mostrar la tabla de todo aquello que incluye el curso

Este componente utiliza la información alojada en el directorio "_data", archivo "instructor", en el objeto "includes-table"

to-learn-course-table.html
